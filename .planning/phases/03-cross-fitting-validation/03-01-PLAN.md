  <files>tests/testthat/test-cross-fitting-leakage.R</files>
  <action>
Create new test file `tests/testthat/test-cross-fitting-leakage.R` with the following tests:

**Test 1: LOSO basis differs from full basis**
- Create fit with S subjects using `.make_fit()` helper pattern from test-dkge-loso.R
- Compare `fit$U` (full basis) with `dkge_loso_contrast()$basis` (held-out basis)
- Verify: held-out basis `U_minus` differs from `fit$U` when subject has non-trivial contribution
- Use Frobenius norm: `norm(U_minus - fit$U, "F") > tolerance` (e.g., 1e-6)
- Test for multiple subjects (not just s=1)

**Test 2: Extreme value injection does not affect held-out basis**
- Create baseline fit
- Get LOSO basis for subject s: `out1 <- dkge_loso_contrast(fit, s, c)`
- Create modified fit with extreme values in subject s only:
  ```r
  fit_extreme <- fit
  fit_extreme$Btil[[s]] <- fit$Btil[[s]] * 1000  # Scale up
  fit_extreme$contribs[[s]] <- compute_contrib(fit_extreme$Btil[[s]], fit$Khalf)  # Recompute contrib
  ```
- Get LOSO basis for same subject: `out2 <- dkge_loso_contrast(fit_extreme, s, c)`
- Verify: `out1$basis` equals `out2$basis` within tight tolerance (1e-10)
- This proves subject s is truly excluded from its own held-out basis

**Test 3: Different subjects get different held-out bases**
- Get LOSO bases for subjects 1, 2, 3
- Verify: U_minus_1, U_minus_2, U_minus_3 are all different from each other
- Use pairwise Frobenius norm comparisons

Include helper function for recomputing subject contribution:
```r
.recompute_contrib <- function(Btil_s, Khalf) {
  right <- Btil_s %*% t(Btil_s)
  S_s <- Khalf %*% right %*% Khalf
  (S_s + t(S_s)) / 2
}
```
