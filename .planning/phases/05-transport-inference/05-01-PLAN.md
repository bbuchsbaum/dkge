  <action>
Create a new test file `tests/testthat/test-sinkhorn-convergence.R` with tests covering:

1. **Doubly-stochastic verification** (core requirement):
   - Test that `rowSums(plan) == mu` within tolerance 1e-6
   - Test that `colSums(plan) == nu` within tolerance 1e-6
   - Test that all entries `plan >= 0`
   - Use multiple scenarios: uniform weights, non-uniform weights, different dimensions

2. **Identity transport for medoid**:
   - Verify `plans[[medoid]] == diag(Q)` exactly when medoid is reference
   - Test with different medoid indices (1, middle, last subject)

3. **Deterministic/identity transport cases**:
   - When source and target embeddings are identical, verify plan concentrates on diagonal
   - Use small epsilon (1e-4) to get near-deterministic behavior
   - Check diagonal entries sum to > 0.9 (sparsity threshold from research)

4. **R/C++ equivalence**:
   - Compare `.dkge_sinkhorn_plan()` (which calls C++) with direct R implementation
   - Since only C++ exists, compare `dkge_transport_to_medoid_sinkhorn` with `dkge_transport_to_medoid_sinkhorn_cpp`
   - Verify results match within 1e-10 tolerance (parallel tolerance from research)

5. **Edge cases**:
   - Very small epsilon (1e-5) - near-deterministic, should still converge
   - Non-convergence case (very small max_iter=5) - should return best result without error
   - Mismatched mass totals - should error with clear message

Use set.seed(42) for reproducibility. Follow existing test patterns in test-transport.R.
