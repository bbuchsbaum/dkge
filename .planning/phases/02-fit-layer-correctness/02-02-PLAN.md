Add determinism tests verifying dkge_fit() produces identical results with the same seed.

Tests to add:

1. **Basic determinism test** - Same fixture + same call produces identical U, Chat, R, weights
2. **Determinism with different w_method** - Test w_method="mfa_sigma1", "energy", "none" are all deterministic
3. **Determinism across solver types** - Test solver="pooled" is deterministic (note: "jd" solver may have random init, document if non-deterministic)

Pattern from research:
```r
test_that("dkge_fit is deterministic with same seed", {
  withr::local_seed(42)
  fixture <- make_fit_fixture(seed = 100)
  fit1 <- dkge_fit(fixture$betas, fixture$designs, K = fixture$K, rank = 2)

  withr::local_seed(42)
  fixture <- make_fit_fixture(seed = 100)
  fit2 <- dkge_fit(fixture$betas, fixture$designs, K = fixture$K, rank = 2)

  expect_equal(fit1$U, fit2$U)
  expect_equal(fit1$Chat, fit2$Chat)
  expect_equal(fit1$R, fit2$R)
  expect_equal(fit1$weights, fit2$weights)
})
```

Important: The fixture must be regenerated with the same seed because withr::local_seed() only affects calls within the local scope. Generate fixture with explicit seed parameter.

For solver="jd", if it's non-deterministic due to random initialization, document this behavior with a comment but still test that with a fixed jd_init it becomes deterministic.
