## Directory Purposes

**R/:**
- Purpose: Core package implementation
- Contains: All exported functions and internal helpers
- Key files:
  - `dkge-data.R`: Subject/data constructors, main `dkge()` entry point
  - `dkge-fit.R`, `dkge-fit-core.R`: Model fitting engine
  - `dkge-contrast.R`: Cross-fitted contrast computation
  - `dkge-transport.R`: Optimal transport to medoid
  - `dkge-inference.R`: Statistical inference
  - `dkge-predict.R`: Out-of-sample prediction
  - `design-kernel.R`: Kernel construction utilities

**src/:**
- Purpose: C++ acceleration via Rcpp/RcppArmadillo
- Contains: Performance-critical computations
- Key files:
  - `sinkhorn.cpp`: Log-domain Sinkhorn OT solver with warm-start
  - `pwdist.cpp`: Pairwise distance computations
  - `RcppExports.cpp`: Auto-generated Rcpp bindings

**tests/testthat/:**
- Purpose: Comprehensive test suite
- Contains: Unit tests, integration tests, regression tests
- Key files:
  - `helper-toy.R`: Synthetic data generators for testing
  - `helper-fit-fixture.R`: Shared fit objects
  - `test-contrast.R`, `test-dkge-loso.R`, `test-dkge-kfold.R`: Cross-fitting tests
  - `test-transport.R`, `test-mapper.R`: Transport layer tests

**vignettes/:**
- Purpose: Long-form documentation and tutorials
- Contains: Rmd files covering workflows, concepts, comparisons
- Key files:
  - `dkge-workflow.Rmd`: End-to-end analysis walkthrough
  - `dkge-design-kernels.Rmd`: Kernel construction guide
  - `dkge-contrasts-inference.Rmd`: Statistical inference patterns
  - `dkge-classification.Rmd`: Decoding analysis guide

**future/:**
- Purpose: Experimental code not yet production-ready
- Contains: Streaming, mixed-effects, advanced transport
- Key files:
  - `dkge-stream.R`: Streaming two-pass implementation
  - `dkge-mixed.R`: Mixed-effects extensions
  - `dkge-neuroim2-stream.R`: On-disk data integration

**data-raw/:**
- Purpose: Algorithm specification and design documents
- Contains: Mathematical foundation
- Key files:
  - `algo.md`: Complete DKGE algorithm specification (white paper)

## Naming Conventions

**Files:**
- `dkge-*.R`: Core package modules (feature-based organization)
- `design-kernel.R`: Standalone kernel utilities
- `hyperdesign-generics.R`: Generic function definitions
- `zzz.R`: Package load hooks
- `globals.R`: Global variable declarations for R CMD check

**Functions:**
- `dkge_*`: Public API functions (exported)
- `.dkge_*`: Internal helpers (not exported, dot prefix)
- `fit_mapper`, `apply_mapper`, `predict_mapper`: S3 generics for mapper system
- `as_dkge_*`: Coercion functions

**Classes:**
- `dkge`: Main fit object
- `dkge_data`: Input bundle
- `dkge_subject`: Single subject record
- `dkge_contrasts`: Contrast results
- `dkge_inference`: Inference results
- `dkge_weights`: Weight specification
- `dkge_folds`: Fold definitions
- `dkge_mapper_*`: Mapper specifications and fits

**Tests:**
- `test-*.R`: Test files named after the feature being tested
- `helper-*.R`: Shared test infrastructure

## Where to Add New Code

**New Feature (e.g., new cross-fitting strategy):**
- Primary code: `R/dkge-{feature}.R`
- Tests: `tests/testthat/test-{feature}.R`
- Update exports in function roxygen2 block with `@export`
- Run `devtools::document()` to update NAMESPACE

**New S3 Method for Existing Generic:**
- Implementation: Same file as the generic or a dedicated `dkge-{class}.R`
- Declaration: `#' @export` in roxygen block
- Pattern: `method.class <- function(x, ...) { ... }`

**New Component/Module:**
- Implementation: `R/dkge-{module}.R`
- Follow existing patterns: internal helpers with `.dkge_*` prefix
- Export public functions via roxygen2 `@export`
- Add tests in `tests/testthat/test-{module}.R`

**New C++ Code:**
- Implementation: `src/{module}.cpp`
- Use RcppArmadillo for linear algebra
- Export via `// [[Rcpp::export]]` attribute
- Run `Rcpp::compileAttributes()` to update `src/RcppExports.cpp` and `R/RcppExports.R`

**Utilities:**
- Shared helpers: `R/dkge-utils.R` (general) or domain-specific file
- Test utilities: `tests/testthat/helper-*.R`

**New Vignette:**
- Location: `vignettes/dkge-{topic}.Rmd`
- Add to `_pkgdown.yml` articles section

- Purpose: Work-in-progress implementations

- Generated: Yes (by GSD commands)
- Committed: Typically no (add to .gitignore)
